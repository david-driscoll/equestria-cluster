# Shared Kustomize Components

> Last updated: 2026-05-24

Kustomize Components are reusable patch sets applied across multiple apps. They are declared in `kubernetes/components/` and referenced from app `ks.yaml` files via the `components:` key.

## Components Overview

```

‚îú‚îÄ‚îÄ alerts/          # Prometheus alert rules
‚îú‚îÄ‚îÄ failover/        # High-availability pod policies
‚îî‚îÄ‚îÄ secret-store/    # ClusterSecretStore for ExternalSecret
```

## How Components Work

Each component is a `kind: Component` Kustomization that:
1. Adds its resources (YAML files)
2. Applies strategic merge patches to matching HelmReleases or Ingresses
3. Makes substitution variables available via `postBuild`

```yaml
# kubernetes/apps/equestria/home/n8n/ks.yaml
components:
  - ../../../../components/postgres
  - ../../../../components/volsync
  - ../../../../components/tailscale
  - ../../../../components/ingress/internal
```

Multiple components compose cleanly ‚Äî each adds its patches without conflict.

## common/ ‚Äî Cluster-Wide Configuration

**Location:** `kubernetes/components/common/`

Automatically injected into every child Kustomization by the root `cluster-apps` Kustomization (no opt-in needed).

**What it provides:**

| File | Purpose |
|------|---------|
| `cluster-secrets.sops.yaml` | Encrypted: `ROOT_DOMAIN`, `INTERNAL_DOMAIN`, `EXTERNAL_DOMAIN`, IP addresses, API tokens |
| `shared-secrets.sops.yaml` | Shared credentials used across apps |
| `sops-age.sops.yaml` | Age key for Flux SOPS decryption |
| `namespace.yaml` | `flux-system` namespace |
| `service-account.yaml` | Flux service account |
| `middleware.yaml` | Traefik middleware (generated by `Update.cs`) |

**ConfigMap generated from `versions.env`:**
- `cluster-versions` ‚Äî `KUBERNETES_VERSION`, `TALOS_VERSION` for substitution

**HelmRelease defaults patched onto all HelmReleases:**
- `maxHistory: 3`, `interval: 15m`, `timeout: 10m`
- `install.remediation.retries: 7`
- `upgrade.remediation.strategy: rollback`
- `decryption.provider: sops`
- `postBuild.substituteFrom: [cluster-secrets, cluster-versions, shared-secrets]`

## ingress/ ‚Äî Ingress Pattern Variants

### ingress/internal ‚Äî Internal Network Only

Exposes the app only to the internal network via `${INTERNAL_DOMAIN}`.

```yaml
components:
  - ../../../../components/ingress/internal
```

**What it patches:**
```yaml
# On matching Ingress resources:
annotations:
  external-dns.alpha.kubernetes.io/target: "${INTERNAL_DOMAIN}"
  traefik.ingress.kubernetes.io/router.entrypoints: websecure
```

**Result:** App reachable at `<app>.<INTERNAL_DOMAIN>`, internal DNS only.

### ingress/authenticated ‚Äî SSO-Protected

Requires Authentik login before access.

```yaml
components:
  - ../../../../components/ingress/authenticated
```

**What it patches:** Same as internal plus Authentik forward-auth middleware.

**Result:** Unauthenticated requests redirected to Authentik login.

### ingress/external ‚Äî Public Internet

Exposes app publicly via Cloudflare DNS.

```yaml
components:
  - ../../../../components/ingress/external
```

**What it patches:**
```yaml
annotations:
  external-dns.alpha.kubernetes.io/target: "${EXTERNAL_DOMAIN}"
  cert-manager.io/cluster-issuer: letsencrypt-production
```

**Result:** App accessible at `<app>.<EXTERNAL_DOMAIN>` with TLS from Let's Encrypt.

## postgres/ ‚Äî Database User Provisioning

**Location:** `kubernetes/components/postgres/`

**When to add:** App needs a PostgreSQL database.

```yaml
components:
  - ../../../../components/postgres
postBuild:
  substitute:
    POSTGRES_NAME: <app>-postgres  # optional, defaults to ${APP}
```

**What happens when you run `task update`:**

1. `Update.cs` scans all `ks.yaml` files for the postgres component reference
2. For each detected app, generates:
   - A CloudNative-PG role in `kubernetes/apps/database/postgres/app/resources/values.yaml`
   - A per-user YAML file in `kubernetes/apps/database/postgres/app/users/<app>-postgres.yaml`
   - An encrypted password entry in `kubernetes/apps/database/postgres/app/passwords.sops.yaml`
3. Passwords are auto-generated (UUID) on first detection and **preserved** on subsequent runs

**Generated secret structure:**
```yaml
# passwords.sops.yaml (encrypted at rest)
apiVersion: v1
kind: Secret
metadata:
  name: <app>-postgres-password
stringData:
  username: <app>-postgres
  password: "auto-generated-uuid"
  database: <app>
  hostname: postgres-rw.database.svc.cluster.local
  port: "5432"
```

**Referencing in ExternalSecret:**
```yaml
dataFrom:
  - extract:
      key: '${APP}-postgres'
    sourceRef:
      storeRef:
        kind: ClusterSecretStore
        name: database
```

**Files never edit directly** ‚Äî always run `task update` after adding/removing postgres component.

## volsync/ ‚Äî PVC Backup to Backblaze B2

**Location:** `kubernetes/components/volsync/`

**When to add:** App has stateful PVCs that need offsite backup.

```yaml
components:
  - ../../../../components/volsync
postBuild:
  substitute:
    VOLSYNC_ACCESSMODES: ReadWriteMany   # or ReadWriteOnce
    VOLSYNC_PUID: "1000"
    VOLSYNC_PGID: "1000"
    # Optional overrides (defaults shown):
    # VOLSYNC_COPYMETHOD: Snapshot
    # VOLSYNC_STORAGECLASS: longhorn-snapshot
    # VOLSYNC_CACHE_CAPACITY: 2Gi
```

**What it creates:**

- `ReplicationSource` ‚Äî daily snapshot at 14:00 UTC, restic to Backblaze B2
- `ReplicationDestination` ‚Äî restore target (used for recovery)
- `PersistentVolumeClaim` ‚Äî the source PVC named `${APP}`

**Backup retention:**
```
hourly: 0, daily: 7, weekly: 4, monthly: 4, yearly: 1, last: 5
```

**Restore procedure:** Create a `ReplicationDestination` from the backup, then swap PVC reference in HelmRelease.

## tailscale/ ‚Äî Tailnet Service Discovery

**Location:** `kubernetes/components/tailscale/`

**When to add:** App should be accessible via Tailscale VPN.

```yaml
components:
  - ../../../../components/tailscale
postBuild:
  substitute:
    TAILSCALE_HOST: <hostname>  # accessible as <hostname>.${TAILSCALE_DOMAIN}
```

**What it creates:**
- A Kubernetes Service with `tailscale.com/tailnet-fqdn` annotation
- App accessible at `<hostname>.<TAILSCALE_DOMAIN>` from any Tailscale-connected device

**Note:** External nodes (Proxmox, dockge hosts) use a different mechanism ‚Äî `ExternalName` Services generated by `kubernetes/apps/tailscale-system/services/Update.cs`. See [`INTEGRATIONS.md`](./INTEGRATIONS.md#6-tailscale-node-map).

## repos/ ‚Äî Helm Chart References

**Location:** `kubernetes/components/repos/`

Contains OCIRepository definitions for Helm charts shared across apps.

**app-template** ‚Äî The primary chart used by ~90% of all apps:
```yaml
# kubernetes/components/repos/app-template/
apiVersion: source.toolkit.fluxcd.io/v1
kind: OCIRepository
metadata:
  name: app-template
spec:
  url: oci://ghcr.io/bjw-s/helm-charts/app-template
```

Referenced in `ks.yaml` when any app uses the app-template chart:
```yaml
components:
  - ../../../../components/repos/app-template
```

## alerts/ ‚Äî Prometheus Alert Rules

**Location:** `kubernetes/components/alerts/`

Injects namespace-wide PrometheusRule resources. Applied at the namespace level (in namespace `kustomization.yaml`), not per-app.

```yaml
# kubernetes/apps/equestria/kustomization.yaml
components:
  - ../../components/alerts
```

## failover/ ‚Äî High-Availability Pod Policies

**Location:** `kubernetes/components/failover/`

### failover/fast-node-eviction

Patches Deployments to evict pods quickly when a node becomes unreachable (5s tolerance vs. default 5min).

```yaml
components:
  - ../../../../components/failover/fast-node-eviction
```

Used by stateless or easily-restartable apps (e.g., n8n).

---

## secret-store/ ‚Äî ExternalSecret Backend

**Location:** `kubernetes/components/secret-store/`

Configures the `ClusterSecretStore` pointing to 1Password Connect. Applied at namespace level.

```yaml
# secretstore.yaml
kind: ClusterSecretStore
metadata:
  name: onepassword-connect
spec:
  provider:
    onepassword:
      connectHost: <1password-connect-service>
      vaults:
        Eris: "1"
```

## Composing Multiple Components

Components can be freely combined:

```yaml
# Full-featured app: postgres + backup + tailnet + internal ingress
components:
  - ../../../../components/postgres
  - ../../../../components/volsync
  - ../../../../components/tailscale
  - ../../../../components/ingress/internal
  - ../../../../components/failover/fast-node-eviction
```

```bash
# Find all apps using tailscale
grep -r "components/tailscale" kubernetes/apps/ --include="*.yaml" -l

# Preview what patches a component applies
flux-local test --path ./kubernetes/flux/cluster --all-namespaces
```

## Debugging Component Patches

```bash
# Check if a component's patch was applied
flux get kustomization <app-name> -o yaml

# See rendered manifests
flux-local build kustomization <app-name>
```
