## Key Safety Rules

- **NEVER commit unencrypted secrets** or `age.key`
- **ALWAYS run `flux-local test`** before pushing any `kubernetes/` changes
- **Do not edit files generated by `Update.cs`** — run `task update` instead
- **Preserve `postBuild` substitution sources** in child kustomizations
- **This is a production cluster** — validate all changes carefully
