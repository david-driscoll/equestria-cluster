## What This Repo Is

A production Talos Linux + Flux GitOps cluster for the "equestria" homelab. Flux watches this repo and reconciles all Kubernetes resources automatically. The cluster uses SOPS+age for secrets, Cloudflare for external DNS/tunnels, and 1Password Connect for dynamic secret injection.

## Common Commands

```sh
# Install all dev tools (pinned versions in .mise.toml)
mise install

# Validate Flux kustomizations locally — ALWAYS run before pushing kubernetes/ changes
flux-local test --enable-helm --all-namespaces --path ./kubernetes/flux/cluster -v

# Force Flux to resync from git
task reconcile

# Run C# update scripts (handles SOPS re-encryption automatically)
task update
# or: dotnet run .mise/tasks/do-update.cs

# Talos operations
task talos:generate-config
task talos:apply-node IP=<node-ip> MODE=auto
task talos:upgrade-node IP=<node-ip>
task talos:upgrade-k8s
task talos:reset

# Bootstrap (initial cluster setup only)
task bootstrap:talos
task bootstrap:apps
```

## Architecture

### GitOps Flow

`kubernetes/flux/cluster/ks.yaml` is the root Flux Kustomization. It injects SOPS decryption and `postBuild` substitution variables (from `cluster-secrets` and `shared-secrets`) into all child Kustomizations. Child Kustomizations must never remove these substitution sources.

### App Directory Pattern

Every app lives at `kubernetes/apps/<NAMESPACE>/<APP>/`:
- `ks.yaml` — Flux Kustomization with `dependsOn`, component references, and postBuild vars (`APP`, `NAMESPACE`, app-specific)
- `kustomization.yaml` — lists resources for the app
- `helmrelease.yaml` — Helm chart deployment (usually using `app-template` OCIRepository)
- `*.sops.yaml` — encrypted secrets (never commit unencrypted)

Namespace grouping: `kubernetes/apps/<NAMESPACE>/kustomization.yaml` lists all apps and includes shared components (`alerts`, `common`, `repos/app-template`).

### Shared Components (`kubernetes/components/`)

| Component | Purpose |
|-----------|---------|
| `common` | Shared cluster vars (`CLUSTER_DOMAIN`, `INTERNAL_DOMAIN`, Tailscale IPs), SOPS secret holders |
| `ingress/internal` | Internal-only ingress annotations/values |
| `ingress/authenticated` | SSO-gated ingress (Authentik) |
| `ingress/external` | Public ingress via external-dns |
| `volsync` | PVC replication and Backblaze offsite backup templates |
| `postgres` | CloudNative-PG database/user provisioning templates |
| `tailscale` | Tailscale operator and IDP integration |
| `repos/app-template` | OCI repo for the `app-template` Helm chart |

Add a component to an app by referencing it in `ks.yaml`: `- ../../../../components/ingress/internal`

### Secrets

- All secrets are `*.sops.yaml` encrypted with age. `age.key` is git-ignored.
- `kubernetes/components/common/cluster-secrets.sops.yaml` and `shared-secrets.sops.yaml` — cluster-wide shared secrets.
- Per-app secrets go in `kubernetes/apps/<NAMESPACE>/<APP>/secret.sops.yaml`.
- Many secrets are provisioned dynamically via ExternalSecret pulling from 1Password Connect (`op://Eris/...`).

### C# Update Scripts

`kubernetes/apps/*/*/Update.cs` scripts generate or regenerate derived resources (volsync CRs, DB provisioning, secret syncs). They handle SOPS encryption transparently. Always use `task update` to run them rather than editing generated files directly.

### Custom Application CRD

`kubernetes/apps/observability/crds/application-crd.yaml` defines a custom `Application` resource. These are discovered by Pulumi to create Authentik apps, OAuth configs, and proxy configurations automatically.

## Critical Safety Rules

- **NEVER commit unencrypted secrets** or `age.key`.
- **ALWAYS run `flux-local test`** before pushing any `kubernetes/` changes.
- **Do not edit files generated by `Update.cs`** — run `task update` instead.
- **NEVER cancel long-running bootstrap/upgrade commands** — they take 45+ minutes.
- **Preserve `postBuild` substitution sources** in child kustomizations.
- This is a **production cluster** — validate all changes carefully.

## Automation

- **Renovate** (`.github/renovate.json5`): automated Helm chart and image updates via PRs.
- **GitHub Actions** (`.github/workflows/flux-local.yaml`): runs `flux-local` diff on all `kubernetes/` PRs.
- **Husky pre-commit**: runs dotnet husky tasks before commits.
